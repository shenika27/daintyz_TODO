# 변경 이력

이 파일의 각 버전 섹션(`## [버전]`)이 GitHub Release 의 패치노트로 자동 반영됩니다.
새 버전을 낼 때 맨 위에 해당 버전 섹션을 추가하고 VERSION 파일을 올리면 됩니다.

## [0.4.8] - 2026-07-01
### 추가
- **인앱 자동 업데이트**: 설정 → 앱 업데이트에서 새 버전 확인·다운로드·재시작
- 앱 시작 시 백그라운드로 새 버전 자동 확인 (있으면 알림)
- 설정에 **패치노트** 버튼 — 최신 릴리즈의 변경 내용을 팝업으로 표시

### 개선
- GitHub Actions 릴리즈 자동화: VERSION 변경 push 시 EXE 빌드·릴리즈 자동 생성
- 설치 위치에 쓸 수 없으면(설치판 등) 릴리즈 페이지로 안내 (조용한 실패 방지)
- 한글 사용자명 경로에서도 업데이트 교체가 정상 동작하도록 처리
