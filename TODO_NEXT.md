## 2. 수익화 — 테마 프리셋 판매 (확정 · 미착수)

캐릭터 이미지 테마 세트(프리셋)를 라이선스키로 판매. **온라인 활성화 방식.**

### 전달 방식 (B안 확정)
- 앱엔 **무료/기본 테마만 내장**(AES-GCM 암호화, 이미 구현됨). 유료 테마는 앱에 넣지 않음.
- 유료 테마는 **결제·활성화 후 Cloudflare R2에서 다운로드** → 미구매자 디스크에 암호문조차 없음.

### 활성화 흐름
1. 네이버스토어 등 결제 → 메일로 라이선스키 발급
2. 앱에 키 입력 → 앱이 `{license_key, machine_id}`를 **Cloudflare Worker**로 전송
3. Worker → **D1** 조회: 키 유효/환불·취소 여부/기기 바인딩(활성화 대수 한도)
4. OK → D1에 `(키 ↔ machine_id, 활성시각)` 기록 → Worker가 **R2 서명 URL(+팩 복호화 키)** 응답
5. 앱이 R2에서 암호화 팩 다운로드 → **machine_id 파생 키로 로컬 재암호화** 저장 → 이후 오프라인 사용

### 앱측 구현 항목
- [ ] `asset_pack`에 **online 키모드 복호화** 추가 (현재 `KEY_MODE_ONLINE`은 거부 스텁)
- [ ] 다운로드 팩 **탐색/저장 경로**(`%APPDATA%\CharacterTodo\packs\`) + 로컬 재암호화 캐시
- [ ] **활성 테마 오버레이**(유료 테마가 기본 이미지 덮어쓰기) + 테마 선택 UI
- [ ] 라이선스키 **입력 UI** + 활성화 클라이언트(HTTPS → Worker)
- [ ] `machine_id` 생성(하드웨어 조합 해시 등)

### 백엔드 구현 항목 (Cloudflare)
- [ ] **Worker**: 활성화 API (키 검증 → D1 → R2 서명 URL/키 발급), 재검증 엔드포인트
- [ ] **D1** 스키마:
  - `licenses(license_key PK, product, status[active/refunded/revoked], max_activations, created_at)`
  - `activations(license_key FK, machine_id, activated_at, last_seen)`
- [ ] **R2**: 암호화 테마 팩 저장 + 시간제한 서명 URL 발급
